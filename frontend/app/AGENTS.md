# Project Guidelines

## Code Style
- Use TypeScript/TSX with existing formatting (single quotes, trailing commas, no semicolons).
- Follow route-file patterns in `src/routes/` and React patterns in `src/main.tsx`.

## Architecture
- App is a Vite + React shell using TanStack file-based routing.
- `src/routes/games/mustachio.tsx` integrates `@parkerstovall.com/mustachio`.
- `src/routes/games/pac-man.tsx` integrates `@parkerstovall.com/pac-man`.

## Build and Test
- From `frontend/`:
  - `pnpm --filter @parkerstovall.com/app dev`
  - `pnpm --filter @parkerstovall.com/app build`
  - `pnpm --filter @parkerstovall.com/app serve`
  - `pnpm --filter @parkerstovall.com/app lint`
  - `pnpm --filter @parkerstovall.com/app test`

## Project Conventions
- Do not manually edit `src/routeTree.gen.ts` (generated by TanStack Router).
- Keep route-level code splitting behavior configured in `vite.config.js`.
- `defaultPreload: 'intent'` in `src/main.tsx` can preload route chunks on user intent.

## Integration Points
- Depends on workspace libraries `@parkerstovall.com/mustachio` and `@parkerstovall.com/pac-man`.
- `phaser` is optimized via `optimizeDeps.include` in `vite.config.js`.

## Security
- No server-side auth logic in this package; keep browser-only assumptions intact.
